# Architecture

## Overview
This project provides examples of how to integrate Mapbox Directions API with the TollGuru Toll API. It allows developers to calculate toll costs for routes generated by Mapbox.

## Data Flow
1. **Request Route**: The application sends a request to the Mapbox Directions API with source and destination coordinates.
2. **Receive Polyline**: Mapbox returns a route which includes a geometry polyline.
3. **Request Tolls**: The application sends the polyline and vehicle details to the TollGuru `complete-polyline-from-mapping-service` endpoint.
4. **Receive Tolls**: TollGuru returns detailed toll information, including rates for different payment methods and vehicle types.

## Major Services
- **Mapbox Directions API**: Used for route calculation and polyline generation.
- **TollGuru Toll API**: Used for calculating tolls based on the route polyline.

## Datastore Choices
- This is a stateless integration example and does not require a local datastore.

## Queues/Jobs
- None.

## Third-party Dependencies
- See `package.json` in the respective language directories (e.g., `javascript/package.json`).

## Auth Model
- **Mapbox**: Access Token (passed as a query parameter).
- **TollGuru**: API Key (passed in the `x-api-key` header).

## Tenancy Model
- **Individual Tenancy**: Each request is authenticated individually. No cross-tenant data sharing occurs as the service is stateless.
